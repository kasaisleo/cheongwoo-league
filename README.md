# 마포 청우회 리그

테니스 클럽 매치 레이팅 웹앱. Next.js (App Router) + Tailwind CSS + Supabase.

## 시작하기

1. 의존성 설치
   ```bash
   npm install
   ```

2. Supabase 프로젝트 생성 후 `.env.example`을 `.env.local`로 복사하고 값 채우기
   ```bash
   cp .env.example .env.local
   ```

3. DB 스키마 적용 (Supabase SQL Editor에 `supabase/migrations/0001_init.sql`,
   `0002_roles_and_contact.sql`, `0003_match_guests_tiebreak.sql` 내용을 순서대로 실행)

4. 개발 서버 실행
   ```bash
   npm run dev
   ```

## 운영진 인증

회원 전체 로그인은 없습니다. 경기 결과 입력(`/matches/new`), 회원 등록(`/members/new`),
게스트 등록(`/guests/new`), 게스트의 정회원 전환은 운영진 비밀번호로 보호되며, `/admin`에서 로그인합니다.
비밀번호는 `.env.local`의 `ADMIN_PASSWORD`로 설정합니다.

## 경기 결과 입력 (청팀 / 우팀)

- 팀 명칭은 "청팀" / "우팀"으로 표시됩니다 (내부 데이터상 A팀/B팀에 대응).
- 각 선수 슬롯은 회원탭/게스트탭을 전환해서 선택할 수 있고, 게스트탭에서 바로 새 게스트를
  등록(이름/나이/구력/휴대폰)해 그 자리에서 선수로 선택할 수 있습니다.
- 세트 스코어는 0~7점입니다. 7-6 스코어가 되면 타이브레이크 입력란이 나타나며,
  타이브레이크 점수는 제한 없이 입력할 수 있습니다.

## ELO 레이팅 계산 방식

- 복식 경기: 팀 레이팅 = 두 선수 레이팅의 평균
- 표준 ELO 공식으로 팀 단위 승/패에 따른 변동치를 계산 (K-factor = 32)
- 같은 팀 두 선수는 동일한 변동치(±)를 적용받음
- 게스트가 경기에 참여하면 임시 레이팅(1300, C급 기준)으로 계산에만 참여하고,
  게스트 본인에게는 레이팅이 저장되지 않습니다. 승/패만 게스트 기록에 반영됩니다.
- 계산 로직: `lib/elo.ts`, 저장/적용 로직: `app/api/matches/route.ts`

## 등급별 초기 레이팅

| 등급 | 초기 레이팅 |
|------|------------|
| A    | 1700       |
| B    | 1500       |
| C    | 1300       |
| D    | 1100       |

## 직책(role)과 실력 등급(grade)

직책(회장/부회장/총무/경기이사/홍보이사/운영이사/섭외이사/정회원/고문)과
실력 등급(A/B/C/D)은 서로 독립적인 항목입니다. 직책은 클럽 내 역할을, 실력 등급은 레이팅 초기값과
실력 구분을 나타냅니다.

## 마포구 대회 점수

회원마다 1~10점의 마포구 대회용 점수(`mapo_score`)를 별도로 관리합니다. 이 점수는 클럽 내
ELO 레이팅 계산과 완전히 분리되어 있으며, 레이팅에 영향을 주지 않습니다.

## 게스트 → 정회원 전환

게스트는 별도로 관리되며, 게스트 목록 페이지에서 운영진이 "정회원으로 전환" 버튼을 누르면
닉네임/등급을 입력해 `members` 테이블에 새 회원으로 등록됩니다. 전환된 게스트 레코드는
삭제되지 않고 방문 이력으로 보존되며, 어떤 회원으로 전환됐는지 추적할 수 있습니다.

## 이번 주 게스트

홈 화면에는 이번 주(월~일) 방문일로 등록된 게스트가 자동으로 표시됩니다.
게스트 등록 시 방문일은 캘린더에서 미리 선택할 수 있습니다.

## 폴더 구조

```
app/            라우트 (페이지 + API)
components/     UI 컴포넌트 (기능별 분리)
lib/            ELO 계산, Supabase 클라이언트, 인증 로직, 매치 화면 변환 헬퍼
supabase/       DB 마이그레이션
```
